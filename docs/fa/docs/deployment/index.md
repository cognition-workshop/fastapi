# استقرار

استقرار یک برنامه **FastAPI** نسبتاً آسان است.

## استقرار به چه معناست

**استقرار** یک برنامه به معنای انجام مراحل لازم برای **در دسترس قرار دادن** آن برای **کاربران** است.

برای یک **API وب**، معمولاً شامل قرار دادن آن در یک **ماشین راه دور** با یک **برنامه سرور** است که عملکرد خوب، پایداری و غیره را فراهم می‌کند، تا **کاربران** شما بتوانند به برنامه به طور کارآمد و بدون وقفه یا مشکل **دسترسی** داشته باشند.

این در تقابل با مراحل **توسعه** است، جایی که دائماً کد را تغییر می‌دهید، آن را خراب و تعمیر می‌کنید، سرور توسعه را متوقف و مجدداً راه‌اندازی می‌کنید و غیره.

## استراتژی‌های استقرار

بسته به مورد استفاده خاص شما و ابزارهایی که استفاده می‌کنید، راه‌های مختلفی برای انجام این کار وجود دارد.

شما می‌توانید با استفاده از ترکیبی از ابزارها خودتان یک **سرور مستقر کنید**، از یک **سرویس ابری** استفاده کنید که بخشی از کار را برای شما انجام می‌دهد، یا گزینه‌های ممکن دیگر.

من برخی از مفاهیم اصلی را که احتمالاً باید هنگام استقرار یک برنامه **FastAPI** در نظر داشته باشید نشان خواهم داد (اگرچه بیشتر آنها برای هر نوع برنامه وب دیگری نیز اعمال می‌شوند).

در بخش‌های بعدی جزئیات بیشتری برای در نظر گرفتن و برخی از تکنیک‌ها را خواهید دید. ✨
