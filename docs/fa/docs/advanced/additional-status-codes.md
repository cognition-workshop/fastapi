# کدهای وضعیت اضافی

به طور پیش‌فرض، **FastAPI** پاسخ‌ها را با استفاده از `JSONResponse` برمی‌گرداند و محتوایی که از *عملیات مسیر* خود برمی‌گردانید را در آن `JSONResponse` قرار می‌دهد.

از کد وضعیت پیش‌فرض یا کدی که در *عملیات مسیر* خود تنظیم کرده‌اید استفاده می‌کند.

## کدهای وضعیت اضافی

اگر می‌خواهید کدهای وضعیت اضافی غیر از کد اصلی برگردانید، می‌توانید این کار را با برگرداندن مستقیم یک `Response`، مانند `JSONResponse`، و تنظیم مستقیم کد وضعیت اضافی انجام دهید.

به عنوان مثال، فرض کنید می‌خواهید یک *عملیات مسیر* داشته باشید که امکان به‌روزرسانی آیتم‌ها را فراهم کند و در صورت موفقیت کد وضعیت HTTP 200 "OK" را برگرداند.

اما همچنین می‌خواهید آیتم‌های جدید را بپذیرد. و وقتی آیتم‌ها قبلاً وجود نداشتند، آنها را ایجاد کند و کد وضعیت HTTP 201 "Created" را برگرداند.

برای رسیدن به این هدف، `JSONResponse` را وارد کنید و محتوای خود را مستقیماً در آنجا برگردانید، با تنظیم `status_code` مورد نظر:

{* ../../docs_src/additional_status_codes/tutorial001_an_py310.py hl[4,25] *}

/// warning

وقتی یک `Response` را مستقیماً برمی‌گردانید، مانند مثال بالا، مستقیماً برگردانده خواهد شد.

با یک مدل سریال‌سازی نخواهد شد و غیره.

مطمئن شوید که داده‌های مورد نظر شما را دارد و مقادیر JSON معتبر هستند (اگر از `JSONResponse` استفاده می‌کنید).

///

/// note | جزئیات فنی

شما همچنین می‌توانید از `from starlette.responses import JSONResponse` استفاده کنید.

**FastAPI** همان `starlette.responses` را به عنوان `fastapi.responses` فقط به عنوان یک سهولت برای شما به عنوان برنامه‌نویس فراهم می‌کند. اما بیشتر پاسخ‌های موجود مستقیماً از Starlette می‌آیند. همینطور `status`.

///

## OpenAPI و مستندات API

اگر کدهای وضعیت و پاسخ‌های اضافی را مستقیماً برگردانید، آنها در شمای OpenAPI (مستندات API) گنجانده نخواهند شد، زیرا FastAPI راهی برای دانستن پیشاپیش آنچه قرار است برگردانید ندارد.

اما می‌توانید آن را در کد خود مستند کنید، با استفاده از: [پاسخ‌های اضافی](additional-responses.md){.internal-link target=_blank}.
