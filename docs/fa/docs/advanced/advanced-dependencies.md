# وابستگی‌های پیشرفته

## وابستگی‌های پارامتریزه شده

تمام وابستگی‌هایی که تاکنون دیدیم یک تابع یا کلاس ثابت هستند.

اما ممکن است مواردی وجود داشته باشد که بخواهید بتوانید پارامترهایی را روی وابستگی تنظیم کنید، بدون اینکه مجبور باشید توابع یا کلاس‌های مختلف زیادی تعریف کنید.

تصور کنید می‌خواهیم وابستگی‌ای داشته باشیم که بررسی کند آیا پارامتر query `q` محتوای ثابت خاصی دارد یا خیر.

اما می‌خواهیم بتوانیم آن محتوای ثابت را پارامتریزه کنیم.

## یک نمونه "قابل فراخوانی"

در پایتون راهی وجود دارد که یک نمونه از یک کلاس را "قابل فراخوانی" کنید.

نه خود کلاس (که از قبل قابل فراخوانی است)، بلکه یک نمونه از آن کلاس.

برای این کار، متد `__call__` را تعریف می‌کنیم:

{* ../../docs_src/dependencies/tutorial011_an_py39.py hl[12] *}

در این مورد، این `__call__` چیزی است که **FastAPI** برای بررسی پارامترهای اضافی و زیروابستگی‌ها استفاده خواهد کرد و همین است که برای ارسال مقدار به پارامتر در *تابع عملیات مسیر* شما فراخوانی خواهد شد.

{* ../../docs_src/dependencies/tutorial011_an_py39.py hl[9] *}

در این مورد، **FastAPI** هرگز `__init__` را لمس نخواهد کرد یا به آن اهمیت نخواهد داد، ما مستقیماً در کد خود از آن استفاده خواهیم کرد.

## استفاده از نمونه به عنوان وابستگی

سپس، می‌توانیم از این `checker` در `Depends(checker)` به جای `Depends(FixedContentQueryChecker)` استفاده کنیم، زیرا وابستگی نمونه `checker` است، نه خود کلاس.

و هنگام حل وابستگی، **FastAPI** این `checker` را به صورت زیر فراخوانی خواهد کرد:

```Python
checker(q="somequery")
```

...و هر چیزی که برمی‌گرداند را به عنوان مقدار وابستگی در *تابع عملیات مسیر* ما به عنوان پارامتر `fixed_content_included` ارسال خواهد کرد:
