# هدرهای پاسخ

## استفاده از پارامتر `Response`

شما می‌توانید یک پارامتر از نوع `Response` را در *تابع عملیات مسیر* خود تعریف کنید (همانطور که برای کوکی‌ها انجام می‌دهید).

و سپس می‌توانید هدرها را در آن شیء پاسخ *موقت* تنظیم کنید.

{* ../../docs_src/response_headers/tutorial002.py hl[1, 7:8] *}

و سپس می‌توانید هر شیئی که نیاز دارید را مانند حالت عادی برگردانید (یک `dict`، یک مدل پایگاه داده و غیره).

و اگر `response_model` تعریف کرده باشید، همچنان برای فیلتر و تبدیل شیئی که برگردانده‌اید استفاده خواهد شد.

**FastAPI** از آن پاسخ *موقت* برای استخراج هدرها (همچنین کوکی‌ها و کد وضعیت) استفاده خواهد کرد و آنها را در پاسخ نهایی که شامل مقداری است که برگردانده‌اید و توسط هر `response_model` فیلتر شده، قرار خواهد داد.

همچنین می‌توانید پارامتر `Response` را در وابستگی‌ها تعریف کنید و هدرها (و کوکی‌ها) را در آنها تنظیم کنید.

شما همچنین می‌توانید از `from starlette.responses import Response` یا `from starlette.responses import JSONResponse` استفاده کنید.

**FastAPI** همان `starlette.responses` را به عنوان `fastapi.responses` فقط به عنوان یک سهولت برای شما به عنوان برنامه‌نویس فراهم می‌کند. اما بیشتر پاسخ‌های موجود مستقیماً از Starlette می‌آیند.

و از آنجا که `Response` می‌تواند به طور مکرر برای تنظیم هدرها و کوکی‌ها استفاده شود، **FastAPI** آن را در `fastapi.Response` نیز فراهم می‌کند.

## هدرهای سفارشی

به خاطر داشته باشید که هدرهای اختصاصی سفارشی را می‌توان <a href="https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers" class="external-link" target="_blank">با استفاده از پیشوند 'X-'</a> اضافه کرد.

اما اگر هدرهای سفارشی دارید که می‌خواهید مرورگر کاربر بتواند آنها را ببیند، باید آنها را به پیکربندی CORS خود اضافه کنید (بیشتر بخوانید در [CORS (اشتراک‌گذاری منابع بین مبدأ)](../tutorial/cors.md){.internal-link target=_blank})، با استفاده از پارامتر `expose_headers` که در <a href="https://www.starlette.io/middleware/#corsmiddleware" class="external-link" target="_blank">مستندات CORS از Starlette</a> مستند شده است.
