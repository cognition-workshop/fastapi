# پاسخ - تغییر کد وضعیت

احتمالاً قبلاً خوانده‌اید که می‌توانید یک [کد وضعیت پاسخ](../tutorial/response-status-code.md){.internal-link target=_blank} پیش‌فرض تنظیم کنید.

اما در برخی موارد نیاز دارید کد وضعیت متفاوتی از پیش‌فرض برگردانید.

## مورد استفاده

به عنوان مثال، تصور کنید که می‌خواهید به طور پیش‌فرض کد وضعیت HTTP "OK" `200` را برگردانید.

اما اگر داده‌ها وجود نداشته باشند، می‌خواهید آنها را ایجاد کنید و کد وضعیت HTTP "CREATED" `201` را برگردانید.

اما همچنان می‌خواهید بتوانید داده‌هایی که برمی‌گردانید را با `response_model` فیلتر و تبدیل کنید.

برای این موارد، می‌توانید از یک پارامتر `Response` استفاده کنید.

## استفاده از پارامتر `Response`

شما می‌توانید یک پارامتر از نوع `Response` را در *تابع عملیات مسیر* خود تعریف کنید (همانطور که برای کوکی‌ها و هدرها انجام می‌دهید).

و سپس می‌توانید `status_code` را در آن شیء پاسخ *موقت* تنظیم کنید.

{* ../../docs_src/response_change_status_code/tutorial001.py hl[1,9,12] *}

و سپس می‌توانید هر شیئی که نیاز دارید را مانند حالت عادی برگردانید (یک `dict`، یک مدل پایگاه داده و غیره).

و اگر `response_model` تعریف کرده باشید، همچنان برای فیلتر و تبدیل شیئی که برگردانده‌اید استفاده خواهد شد.

**FastAPI** از آن پاسخ *موقت* برای استخراج کد وضعیت (همچنین کوکی‌ها و هدرها) استفاده خواهد کرد و آنها را در پاسخ نهایی که شامل مقداری است که برگردانده‌اید و توسط هر `response_model` فیلتر شده، قرار خواهد داد.

همچنین می‌توانید پارامتر `Response` را در وابستگی‌ها تعریف کنید و کد وضعیت را در آنها تنظیم کنید. اما به خاطر داشته باشید که آخرین مقدار تنظیم شده برنده خواهد بود.
