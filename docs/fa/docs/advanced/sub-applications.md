# زیربرنامه‌ها - سوار کردن

اگر نیاز دارید دو برنامه مستقل FastAPI داشته باشید، با OpenAPI مستقل خود و رابط‌های مستندات خود، می‌توانید یک برنامه اصلی داشته باشید و یک (یا چند) زیربرنامه را روی آن "سوار" کنید.

## سوار کردن یک برنامه **FastAPI**

"سوار کردن" به معنای افزودن یک برنامه کاملاً "مستقل" در یک مسیر خاص است که سپس مسئولیت مدیریت همه چیز در آن مسیر را بر عهده می‌گیرد، با _عملیات‌های مسیر_ تعریف شده در آن زیربرنامه.

### برنامه سطح بالا

ابتدا، برنامه اصلی، سطح بالا، **FastAPI** و *عملیات‌های مسیر* آن را ایجاد کنید:

{* ../../docs_src/sub_applications/tutorial001.py hl[3, 6:8] *}

### زیربرنامه

سپس، زیربرنامه و *عملیات‌های مسیر* آن را ایجاد کنید.

این زیربرنامه فقط یک برنامه استاندارد FastAPI دیگر است، اما این همان برنامه‌ای است که "سوار" خواهد شد:

{* ../../docs_src/sub_applications/tutorial001.py hl[11, 14:16] *}

### سوار کردن زیربرنامه

در برنامه سطح بالای خود، `app`، زیربرنامه `subapi` را سوار کنید.

در این مورد، در مسیر `/subapi` سوار خواهد شد:

{* ../../docs_src/sub_applications/tutorial001.py hl[11, 19] *}

### بررسی مستندات خودکار API

اکنون، دستور `fastapi` را با فایل خود اجرا کنید:

<div class="termy">

```console
$ fastapi dev main.py

<span style="color: green;">INFO</span>:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
```

</div>

و مستندات را در <a href="http://127.0.0.1:8000/docs" class="external-link" target="_blank">http://127.0.0.1:8000/docs</a> باز کنید.

مستندات خودکار API برای برنامه اصلی را خواهید دید که فقط شامل _عملیات‌های مسیر_ خود آن است:

<img src="/img/tutorial/sub-applications/image01.png">

و سپس، مستندات زیربرنامه را در <a href="http://127.0.0.1:8000/subapi/docs" class="external-link" target="_blank">http://127.0.0.1:8000/subapi/docs</a> باز کنید.

مستندات خودکار API برای زیربرنامه را خواهید دید که فقط شامل _عملیات‌های مسیر_ خود آن است، همه تحت پیشوند زیرمسیر صحیح `/subapi`:

<img src="/img/tutorial/sub-applications/image02.png">

اگر سعی کنید با هر یک از دو رابط کاربری تعامل کنید، به درستی کار خواهند کرد، زیرا مرورگر قادر خواهد بود با هر برنامه یا زیربرنامه خاص ارتباط برقرار کند.

### جزئیات فنی: `root_path`

هنگامی که یک زیربرنامه را همانطور که در بالا توضیح داده شد سوار می‌کنید، FastAPI با استفاده از مکانیزمی از مشخصات ASGI به نام `root_path`، مسیر سوار شده را به زیربرنامه ارتباط می‌دهد.

به این ترتیب، زیربرنامه می‌داند که از آن پیشوند مسیر برای رابط مستندات استفاده کند.

و زیربرنامه همچنین می‌تواند زیربرنامه‌های سوار شده خود را داشته باشد و همه چیز به درستی کار خواهد کرد، زیرا FastAPI تمام این `root_path`ها را به طور خودکار مدیریت می‌کند.

درباره `root_path` و نحوه استفاده صریح از آن در بخش [پشت یک پروکسی](behind-a-proxy.md){.internal-link target=_blank} بیشتر خواهید آموخت.
