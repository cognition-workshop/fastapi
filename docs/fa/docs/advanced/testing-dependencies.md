# تست وابستگی‌ها با بازنویسی

## بازنویسی وابستگی‌ها در هنگام تست

سناریوهایی وجود دارد که ممکن است بخواهید یک وابستگی را در هنگام تست بازنویسی کنید.

نمی‌خواهید وابستگی اصلی اجرا شود (و نه هیچ یک از زیروابستگی‌هایی که ممکن است داشته باشد).

در عوض، می‌خواهید یک وابستگی متفاوت ارائه دهید که فقط در طول تست‌ها (احتمالاً فقط برخی تست‌های خاص) استفاده شود و مقداری را ارائه دهد که می‌تواند در جایی که مقدار وابستگی اصلی استفاده می‌شد، استفاده شود.

### مورد استفاده: سرویس خارجی

یک مثال می‌تواند این باشد که یک ارائه‌دهنده احراز هویت خارجی دارید که باید آن را فراخوانی کنید.

یک توکن به آن ارسال می‌کنید و یک کاربر احراز هویت شده برمی‌گرداند.

این ارائه‌دهنده ممکن است به ازای هر درخواست از شما هزینه دریافت کند و فراخوانی آن ممکن است زمان بیشتری نسبت به داشتن یک کاربر ساختگی ثابت برای تست‌ها بگیرد.

احتمالاً می‌خواهید ارائه‌دهنده خارجی را یک بار تست کنید، اما لزوماً نمی‌خواهید آن را برای هر تستی که اجرا می‌شود فراخوانی کنید.

در این مورد، می‌توانید وابستگی که آن ارائه‌دهنده را فراخوانی می‌کند بازنویسی کنید و از یک وابستگی سفارشی که یک کاربر ساختگی برمی‌گرداند، فقط برای تست‌های خود استفاده کنید.

### استفاده از ویژگی `app.dependency_overrides`

برای این موارد، برنامه **FastAPI** شما یک ویژگی `app.dependency_overrides` دارد، که یک `dict` ساده است.

برای بازنویسی یک وابستگی برای تست، وابستگی اصلی (یک تابع) را به عنوان کلید و بازنویسی وابستگی خود (تابع دیگر) را به عنوان مقدار قرار دهید.

و سپس **FastAPI** آن بازنویسی را به جای وابستگی اصلی فراخوانی خواهد کرد.

/// tip

شما می‌توانید یک بازنویسی وابستگی را برای وابستگی‌ای که در هر جای برنامه **FastAPI** شما استفاده شده تنظیم کنید.

وابستگی اصلی می‌تواند در یک *تابع عملیات مسیر*، یک *دکوراتور عملیات مسیر* (زمانی که از مقدار بازگشتی استفاده نمی‌کنید)، یک فراخوانی `.include_router()` و غیره استفاده شده باشد.

FastAPI همچنان قادر به بازنویسی آن خواهد بود.

///

سپس می‌توانید بازنویسی‌های خود را با تنظیم `app.dependency_overrides` به یک `dict` خالی بازنشانی (حذف) کنید:

```Python
app.dependency_overrides = {}
```

/// tip

اگر می‌خواهید یک وابستگی را فقط در طول برخی تست‌ها بازنویسی کنید، می‌توانید بازنویسی را در ابتدای تست (داخل تابع تست) تنظیم کنید و در انتهای آن (در پایان تابع تست) بازنشانی کنید.
