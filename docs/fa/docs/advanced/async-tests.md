# تست‌های ناهمگام

شما قبلاً دیده‌اید که چگونه برنامه‌های **FastAPI** خود را با استفاده از `TestClient` ارائه شده تست کنید. تا کنون، فقط نحوه نوشتن تست‌های همگام بدون استفاده از توابع `async` را دیده‌اید.

قادر بودن به استفاده از توابع ناهمگام در تست‌ها می‌تواند مفید باشد، برای مثال، هنگامی که به صورت ناهمگام از پایگاه داده خود query می‌زنید. تصور کنید می‌خواهید ارسال درخواست‌ها به برنامه FastAPI خود را تست کنید و سپس تأیید کنید که بک‌اند شما داده‌های صحیح را با موفقیت در پایگاه داده نوشته، در حالی که از یک کتابخانه پایگاه داده ناهمگام استفاده می‌کنید.

بیایید ببینیم چگونه می‌توانیم این کار را انجام دهیم.

## pytest.mark.anyio

اگر بخواهیم توابع ناهمگام را در تست‌های خود فراخوانی کنیم، توابع تست ما باید ناهمگام باشند. AnyIO یک پلاگین مناسب برای این کار فراهم می‌کند که به ما اجازه می‌دهد مشخص کنیم برخی توابع تست به صورت ناهمگام فراخوانی شوند.

## HTTPX

حتی اگر برنامه **FastAPI** شما از توابع `def` معمولی به جای `async def` استفاده کند، همچنان یک برنامه `async` در زیر آن است.

`TestClient` کمی جادو در داخل انجام می‌دهد تا برنامه ناهمگام FastAPI را در توابع تست `def` معمولی شما فراخوانی کند و از pytest استاندارد استفاده کند. اما وقتی از آن در داخل توابع ناهمگام استفاده می‌کنیم، این جادو دیگر کار نمی‌کند. با اجرای تست‌های ما به صورت ناهمگام، دیگر نمی‌توانیم از `TestClient` در داخل توابع تست خود استفاده کنیم.

`TestClient` بر اساس <a href="https://www.python-httpx.org" class="external-link" target="_blank">HTTPX</a> است، و خوشبختانه، می‌توانیم مستقیماً از آن برای تست API استفاده کنیم.

## جزئیات

دکوراتور `@pytest.mark.anyio` به pytest می‌گوید که این تابع تست باید به صورت ناهمگام فراخوانی شود:

{* ../../docs_src/async_tests/test_main.py hl[7] *}

/// tip

توجه کنید که تابع تست اکنون `async def` است به جای فقط `def` مانند قبل هنگام استفاده از `TestClient`.

///

سپس می‌توانیم یک `AsyncClient` با برنامه ایجاد کنیم و با استفاده از `await` درخواست‌های ناهمگام به آن ارسال کنیم.

{* ../../docs_src/async_tests/test_main.py hl[9:12] *}

این معادل:

```Python
response = client.get('/')
```

...است که برای ارسال درخواست‌ها با `TestClient` استفاده می‌کردیم.

/// tip

توجه کنید که ما از async/await با `AsyncClient` جدید استفاده می‌کنیم - درخواست ناهمگام است.

///

/// warning

اگر برنامه شما به رویدادهای lifespan وابسته است، `AsyncClient` این رویدادها را فعال نخواهد کرد. برای اطمینان از فعال شدن آنها، از `LifespanManager` از <a href="https://github.com/florimondmanca/asgi-lifespan#usage" class="external-link" target="_blank">florimondmanca/asgi-lifespan</a> استفاده کنید.

///

## سایر فراخوانی‌های تابع ناهمگام

از آنجا که تابع تست اکنون ناهمگام است، می‌توانید سایر توابع `async` را نیز علاوه بر ارسال درخواست به برنامه FastAPI خود در تست‌هایتان فراخوانی (و `await`) کنید، دقیقاً همانطور که آنها را در هر جای دیگری در کد خود فراخوانی می‌کنید.

/// tip

اگر هنگام یکپارچه‌سازی فراخوانی‌های تابع ناهمگام در تست‌های خود با خطای `RuntimeError: Task attached to a different loop` مواجه شدید (مثلاً هنگام استفاده از <a href="https://stackoverflow.com/questions/41584243/runtimeerror-task-attached-to-a-different-loop" class="external-link" target="_blank">MotorClient مونگو‌دی‌بی</a>)، به یاد داشته باشید که اشیایی که نیاز به event loop دارند را فقط در داخل توابع async نمونه‌سازی کنید، مثلاً یک callback `'@app.on_event("startup")`.

///
