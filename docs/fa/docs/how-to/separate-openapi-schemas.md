# جداسازی شماهای OpenAPI برای ورودی و خروجی

هنگامی که از مدل‌های **Pydantic** برای اعتبارسنجی ورودی و سریال‌سازی خروجی در FastAPI استفاده می‌کنید، ممکن است متوجه شوید که رفتار **شمای OpenAPI** تولید شده بین ورودی و خروجی متفاوت است. بیایید ببینیم چرا و چگونه آن را مدیریت کنیم.

## فیلدهای Pydantic و مقادیر پیش‌فرض

وقتی یک فیلد Pydantic دارای مقدار `default` باشد، در ورودی **الزامی** نیست. اما در خروجی، همیشه شامل خواهد شد.

### مدل ورودی

اگر مدلی مثل این دارید:

```Python
from pydantic import BaseModel

class Item(BaseModel):
    name: str
    description: str | None = None
```

...فیلد `description` در **ورودی** (بدنه درخواست) الزامی نیست، زیرا مقدار پیش‌فرض `None` دارد.

### مدل خروجی

اما وقتی همین مدل برای **خروجی** (پاسخ) استفاده شود، فیلد `description` همیشه در پاسخ JSON وجود خواهد داشت (حتی اگر مقدار آن `None` باشد).

### شمای OpenAPI

به طور پیش‌فرض، FastAPI از **Pydantic v2** استفاده می‌کند که شماهای ورودی و خروجی را به صورت هوشمند جدا می‌کند. فیلدهای با مقادیر پیش‌فرض در شمای **ورودی** به عنوان اختیاری و در شمای **خروجی** به عنوان الزامی علامت‌گذاری می‌شوند.

این رفتار باعث می‌شود مستندات API شما دقیق‌تر باشد و کلاینت‌ها بدانند دقیقاً چه انتظاری از ورودی و خروجی داشته باشند.

### غیرفعال کردن جداسازی شماها

اگر به هر دلیلی نمی‌خواهید این جداسازی انجام شود، می‌توانید آن را غیرفعال کنید:

```Python
app = FastAPI(separate_input_output_schemas=False)
```

/// info

پشتیبانی از جداسازی شماها از FastAPI نسخه `0.102.0` اضافه شده است.
