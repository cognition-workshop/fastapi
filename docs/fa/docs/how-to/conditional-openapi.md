# OpenAPI شرطی

اگر نیاز داشته باشید، می‌توانید از تنظیمات و متغیرهای محیطی برای پیکربندی شرطی OpenAPI بسته به محیط و حتی غیرفعال کردن کامل آن استفاده کنید.

## درباره امنیت، APIها و مستندات

پنهان کردن رابط‌های مستندات شما در محیط پروداکشن *نباید* روش محافظت از API شما باشد.

این هیچ امنیت اضافی به API شما اضافه نمی‌کند، *عملیات‌های مسیر* همچنان در جایی که هستند در دسترس خواهند بود.

اگر یک نقص امنیتی در کد شما وجود داشته باشد، همچنان وجود خواهد داشت.

پنهان کردن مستندات فقط درک نحوه تعامل با API شما را دشوارتر می‌کند و ممکن است اشکال‌زدایی آن در محیط پروداکشن را برای شما دشوارتر کند. می‌توان آن را صرفاً نوعی <a href="https://en.wikipedia.org/wiki/Security_through_obscurity" class="external-link" target="_blank">امنیت از طریق ابهام</a> در نظر گرفت.

اگر می‌خواهید API خود را ایمن کنید، چندین کار بهتر وجود دارد که می‌توانید انجام دهید، برای مثال:

* مطمئن شوید که مدل‌های Pydantic خوب تعریف شده‌ای برای بدنه‌های درخواست و پاسخ‌های خود دارید.
* هر مجوز و نقش مورد نیاز را با استفاده از وابستگی‌ها پیکربندی کنید.
* هرگز رمزهای عبور را به صورت متن ساده ذخیره نکنید، فقط هش رمز عبور.
* ابزارهای رمزنگاری شناخته شده مانند Passlib و توکن‌های JWT و غیره را پیاده‌سازی و استفاده کنید.
* در صورت نیاز، کنترل‌های مجوز دقیق‌تر را با scopeهای OAuth2 اضافه کنید.
* ...و غیره.

با این حال، ممکن است یک مورد استفاده بسیار خاص داشته باشید که واقعاً نیاز به غیرفعال کردن مستندات API برای برخی محیط‌ها (مثلاً برای پروداکشن) یا بسته به پیکربندی‌های متغیرهای محیطی دارید.

## OpenAPI شرطی از تنظیمات و متغیرهای محیطی

شما به راحتی می‌توانید از همان تنظیمات Pydantic برای پیکربندی OpenAPI تولید شده و رابط‌های مستندات استفاده کنید.

برای مثال:

{* ../../docs_src/conditional_openapi/tutorial001.py hl[6,11] *}

در اینجا تنظیم `openapi_url` را با همان پیش‌فرض `"/openapi.json"` تعریف می‌کنیم.

و سپس هنگام ایجاد برنامه `FastAPI` از آن استفاده می‌کنیم.

سپس می‌توانید OpenAPI (از جمله مستندات رابط کاربری) را با تنظیم متغیر محیطی `OPENAPI_URL` به رشته خالی غیرفعال کنید، مانند:

<div class="termy">

```console
$ OPENAPI_URL= uvicorn main:app

<span style="color: green;">INFO</span>:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
```

</div>

سپس اگر به URLهای `/openapi.json`، `/docs` یا `/redoc` بروید، فقط خطای `404 Not Found` دریافت خواهید کرد:

```JSON
{
    "detail": "Not Found"
}
```
