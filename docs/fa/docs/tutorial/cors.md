# CORS (اشتراک‌گذاری منابع بین مبدأها)

<a href="https://developer.mozilla.org/en-US/docs/Web/HTTP/CORS" class="external-link" target="_blank">CORS یا "اشتراک‌گذاری منابع بین مبدأها"</a> به شرایطی اشاره دارد که یک فرانت‌اند در حال اجرا در مرورگر کد جاوااسکریپتی دارد که با یک بک‌اند ارتباط برقرار می‌کند و بک‌اند در "مبدأ" متفاوتی نسبت به فرانت‌اند قرار دارد.

## مبدأ

یک مبدأ ترکیبی از پروتکل (`http`، `https`)، دامنه (`myapp.com`، `localhost`، `localhost.tiangolo.com`) و پورت (`80`، `443`، `8080`) است.

بنابراین، همه اینها مبدأهای مختلفی هستند:

* `http://localhost`
* `https://localhost`
* `http://localhost:8080`

حتی اگر همه در `localhost` هستند، از پروتکل‌ها یا پورت‌های مختلف استفاده می‌کنند، بنابراین "مبدأهای" مختلفی هستند.

## مراحل

بنابراین، فرض کنید فرانت‌اندی در مرورگر شما در `http://localhost:8080` در حال اجراست و جاوااسکریپت آن تلاش می‌کند با بک‌اندی در حال اجرا در `http://localhost` ارتباط برقرار کند (چون پورت مشخص نکرده‌ایم، مرورگر پورت پیش‌فرض `80` را فرض می‌کند).

سپس، مرورگر یک درخواست HTTP `OPTIONS` به بک‌اند `:80` ارسال می‌کند و اگر بک‌اند هدرهای مناسب برای مجاز کردن ارتباط از این مبدأ متفاوت (`http://localhost:8080`) ارسال کند، آنگاه مرورگر `:8080` اجازه می‌دهد جاوااسکریپت در فرانت‌اند درخواست خود را به بک‌اند `:80` ارسال کند.

برای رسیدن به این هدف، بک‌اند `:80` باید لیستی از "مبدأهای مجاز" داشته باشد.

در این مورد، لیست باید شامل `http://localhost:8080` باشد تا فرانت‌اند `:8080` به درستی کار کند.

## Wildcardها

همچنین ممکن است لیست را به عنوان `"*"` (یک "wildcard") اعلان کنید تا بگویید همه مجاز هستند.

اما این فقط انواع خاصی از ارتباط را مجاز می‌کند، به استثنای هر چیزی که شامل اعتبارنامه‌ها باشد: کوکی‌ها، هدرهای Authorization مانند آنهایی که با Bearer Tokenها استفاده می‌شوند و غیره.

بنابراین، برای اینکه همه چیز به درستی کار کند، بهتر است مبدأهای مجاز را صریحاً مشخص کنید.

## استفاده از `CORSMiddleware`

می‌توانید آن را در برنامه **FastAPI** خود با استفاده از `CORSMiddleware` پیکربندی کنید.

* `CORSMiddleware` را وارد کنید.
* لیستی از مبدأهای مجاز (به عنوان رشته) ایجاد کنید.
* آن را به عنوان "middleware" به برنامه **FastAPI** خود اضافه کنید.

همچنین می‌توانید مشخص کنید آیا بک‌اند شما اجازه می‌دهد:

* اعتبارنامه‌ها (هدرهای Authorization، کوکی‌ها و غیره).
* متدهای HTTP خاص (`POST`، `PUT`) یا همه آنها با wildcard `"*"`.
* هدرهای HTTP خاص یا همه آنها با wildcard `"*"`.

پارامترهای پیش‌فرض استفاده شده توسط پیاده‌سازی `CORSMiddleware` به طور پیش‌فرض محدودکننده هستند، بنابراین باید صریحاً مبدأها، متدها یا هدرهای خاص را فعال کنید تا مرورگرها اجازه استفاده از آنها در یک زمینه بین‌دامنه‌ای را داشته باشند.

آرگومان‌های زیر پشتیبانی می‌شوند:

* `allow_origins` - لیستی از مبدأهایی که باید اجازه درخواست‌های بین مبدأ داشته باشند. مثلاً `['https://example.org', 'https://www.example.org']`. می‌توانید از `['*']` برای مجاز کردن هر مبدأ استفاده کنید.
* `allow_origin_regex` - یک رشته regex برای مطابقت با مبدأهایی که باید اجازه درخواست‌های بین مبدأ داشته باشند. مثلاً `'https://.*\.example\.org'`.
* `allow_methods` - لیستی از متدهای HTTP که باید برای درخواست‌های بین مبدأ مجاز باشند. پیش‌فرض `['GET']` است. می‌توانید از `['*']` برای مجاز کردن همه متدهای استاندارد استفاده کنید.
* `allow_headers` - لیستی از هدرهای درخواست HTTP که باید برای درخواست‌های بین مبدأ پشتیبانی شوند. پیش‌فرض `[]` است. می‌توانید از `['*']` برای مجاز کردن همه هدرها استفاده کنید. هدرهای `Accept`، `Accept-Language`، `Content-Language` و `Content-Type` همیشه برای <a href="https://developer.mozilla.org/en-US/docs/Web/HTTP/CORS#simple_requests" class="external-link" rel="noopener" target="_blank">درخواست‌های ساده CORS</a> مجاز هستند.
* `allow_credentials` - نشان می‌دهد که کوکی‌ها باید برای درخواست‌های بین مبدأ پشتیبانی شوند. پیش‌فرض `False` است. همچنین، `allow_origins` نمی‌تواند روی `['*']` تنظیم شود تا اعتبارنامه‌ها مجاز باشند، مبدأها باید مشخص شوند.
* `expose_headers` - هر هدر پاسخی که باید برای مرورگر قابل دسترسی باشد را نشان می‌دهد. پیش‌فرض `[]` است.
* `max_age` - حداکثر زمان به ثانیه برای کش کردن پاسخ‌های CORS توسط مرورگرها را تنظیم می‌کند. پیش‌فرض `600` است.

middleware به دو نوع خاص از درخواست HTTP پاسخ می‌دهد...

### درخواست‌های preflight CORS

اینها هر درخواست `OPTIONS` با هدرهای `Origin` و `Access-Control-Request-Method` هستند.

در این مورد middleware درخواست ورودی را رهگیری می‌کند و با هدرهای CORS مناسب پاسخ می‌دهد، و یک پاسخ `200` یا `400` برای اطلاع‌رسانی.

### درخواست‌های ساده

هر درخواستی با هدر `Origin`. در این مورد middleware درخواست را به صورت عادی عبور می‌دهد، اما هدرهای CORS مناسب را در پاسخ اضافه می‌کند.
