# مدل‌های پارامتر Query

اگر گروهی از **پارامترهای query** مرتبط دارید، می‌توانید یک **مدل Pydantic** برای تعریف آنها ایجاد کنید.

این به شما امکان می‌دهد **مدل را** در **مکان‌های متعدد** **دوباره استفاده کنید** و همچنین اعتبارسنجی‌ها و متادیتا را برای تمام پارامترها به یکباره تعریف کنید. 😎

/// note

این از نسخه `0.115.0` FastAPI پشتیبانی می‌شود. 🤓

///

## ممنوع کردن پارامترهای Query اضافی

در برخی موارد استفاده خاص (احتمالاً خیلی رایج نیست)، ممکن است بخواهید پارامترهای query دریافتی را **محدود** کنید.

می‌توانید از پیکربندی مدل Pydantic برای `forbid` کردن هر فیلد `extra` استفاده کنید:

{* ../../docs_src/query_param_models/tutorial002_an_py310.py hl[10] *}

اگر یک کلاینت سعی کند داده‌های **اضافی** در **پارامترهای query** ارسال کند، پاسخ **خطا** دریافت خواهد کرد.

به عنوان مثال، اگر کلاینت سعی کند یک پارامتر query به نام `tool` با مقدار `plumbus` ارسال کند، مانند:

```http
https://example.com/items/?limit=10&tool=plumbus
```

پاسخ **خطایی** دریافت خواهد کرد که به آنها می‌گوید پارامتر query `tool` مجاز نیست:

```json
{
    "detail": [
        {
            "type": "extra_forbidden",
            "loc": ["query", "tool"],
            "msg": "Extra inputs are not permitted",
            "input": "plumbus"
        }
    ]
}
```
